Stepper

Stepper is a JavaScript library for debgging JavaScript. This may be used to present a simplified and/or consistent debugging interface to novice programmers who are just learning to use a debugger. It might also be used in the future to implement things like event recording and playback which current browser debuggers don't support.

It uses ES6 Generators to which allow flow control to exit a function and then re-enter it at the same place that it left using the yield keyword. For more information see function* [MDN]. I plan on writing some blog posts that describe the inner workings in more detail... stay tuned.

It supports the following features:

  • breakpoints (click to the left of the line number)
  • stepping in/over/out
  • callstack listing
  • local variables
  • inspecting recursive data structures
  • proper timing for events that occur while paused

Source is availble on github.

Live demo

The demo uses processing.js as a library. The are many things that don't work yet, but the biggest one is that you can't pass user defined functions as callbacks to built-in functions... yet (right now the "draw" function is being special cased).
